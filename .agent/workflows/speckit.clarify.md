# /speckit.clarify

Use this command to analyze `specifications.md` and identify areas that need more detail.

## Workflow

1.  **Scan Specs**: Read `specifications.md`.
2.  **Identify Gaps**: Flag missing edge cases, ambiguous requirements, or technical unknowns.
3.  **Ask Questions**: Present a numbered list of clarifying questions to the user.
